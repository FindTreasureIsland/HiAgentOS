# 周报内容生成规则

> 本文档定义公司周报的内容生成规则，用于指导周报的结构、内容和数据来源。

---

## 一、周报结构概览

周报分为以下四个维度：

| 维度 | 数据来源 | 总结方向 |
|------|---------|---------|
| 🔭 市场拓展 | 市场、售前等岗位的工作计划和报告 | 新增市场线索、重点市场机会进度 |
| 📦 交付项目 | 项目经理、产品、技术等岗位的工作计划和报告 | 重点项目进度、需求变更、项目风险 |
| 🏗️ 产品研发 | 产品、技术等岗位的工作计划和报告 | 不同产品的进度、完成工作 |
| 🏢 公司管理 | 公司管理层、行政等岗位的工作计划和报告 | 管理工作进度、完成内容 |

## 四、各维度详细内容规则

### 3.1 市场拓展

**数据来源**：市场、售前等岗位

#### 📌 新增市场线索

收集本周新增的市场机会信息：

| 字段 | 说明 |
|------|------|
| 线索名称 | 项目或机会名称 |
| 来源 | 信息来源（客户介绍、活动获取等） |
| 跟进状态 | 初步接触、深入沟通、投标中等 |

**表格格式**：
```markdown
| 线索名称 | 来源 | 跟进状态 |
|---------|------|---------|
| ... | ... | ... |
```

#### 📌 重点市场机会进度

| 字段 | 说明 |
|------|------|
| 项目 | 项目名称 |
| 客户 | 客户/合作方 |
| 机会描述 | 简要描述机会内容 |
| 本周进展 | 具体做了什么 |
| 状态 | 🟢 跟踪中 / 🟡 需求确认中 / 🟠 投标中 / 🔴 赢单/丢单 |

**风险**：列出当前面临的风险（1-2句话）
**改进**：后续改进行动（1-2句话）

---

### 3.2 交付项目

**数据来源**：项目经理、产品、技术等岗位

#### 📌 重点项目进度

| 字段 | 说明 |
|------|------|
| 项目 | 项目名称 |
| 客户 | 客户名称（内部项目填"内部"） |
| 本周完成 | 具体完成的工作 |
| 进行中 | 正在进行的工作 |
| 阻塞 | 阻塞问题（无则填"-"） |

**表格格式**：
```markdown
| 项目 | 客户 | 本周完成 | 进行中 | 阻塞 |
|------|------|---------|-------|------|
| ... | ... | ... | ... | ... |
```

#### 📌 需求变更

| 字段 | 说明 |
|------|------|
| 项目 | 变更所属项目 |
| 变更内容 | 变更描述 |
| 影响 | 对进度/成本的影响 |
| 处理方式 | 已采取的措施 |

**若无需求变更**：注明"本周无重大需求变更"

#### 📌 项目风险

| 字段 | 说明 |
|------|------|
| 项目 | 风险所属项目 |
| 风险描述 | 风险内容 |
| 等级 | 🔴 高 / 🟠 中 / 🟡 低 |
| 应对措施 | 准备采取的措施 |

**风险**：总体风险概述（1-2句话）
**改进**：后续改进行动（1-2句话）

## 六、周报生成流程

1. **数据收集**
   - 读取所有人员的日计划文件（周一至周五）
   - 按四个维度分类整理

2. **内容生成**
   - 按照上述规则填充各维度内容
   - 确保数据准确，不遗漏

3. **审核要点**
   - 检查完成率是否合理
   - 确认阻塞问题是否都有解决方案
   - 确认本周重点是否覆盖高优先级任务

4. **输出**
   - 生成详细版（Markdown文档）
   - 生成简洁版（飞书群消息）
